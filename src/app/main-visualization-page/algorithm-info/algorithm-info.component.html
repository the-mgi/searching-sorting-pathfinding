Linear Search

What is? Linear Search is the most basic type of Sequential Search which can be used to search objects in any given array. It doesn't care if the given array is in sorted order or not, it gives the accurate results in both cases. It can be useful in areas with much less array sizes because of it's time complexity.

How? It achieves it's desired goal by traversing each element of the given array and checking for if the given key matches the element.

Asymptotic Complexity Analysis

Worst Case? Worst time complexity is denoted by O and is equals to O(n) as n denotes the number of total elements in the array. Worst Complexity would be that if the element is not present in the array and traversing the complete array would be of no use.

Average Case? Average time complexity, denoted by θ and is equals to θ(n) as n denotes the number of total elements in the array. Average Complexity would be that element could be present at any index in the array or could be that not present in the array!

Best Case? Best time complexity, denoted by Ω and is equals to Ω( 1 ) . Best Complexity would be if the required element is present at first index of the array and the algorithm would be able to get that in only 1 iteration.

Space Complexity

Space Complexity Space Complexity, denoted by O . Linear Search Iterative does not keep the elements in memory, thereby unlike recursion its Space Complexity would be O( 1 )
